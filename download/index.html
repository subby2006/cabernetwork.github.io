Download

Windows

  1. Visit Cabernet Releases on GitHub.
  2. Download the appropriate release for Windows.
  3. Follow the installation steps provided in the GitHub repository or the release notes.

Linux

Requirements:

  • Python 3.8+
  • Python Modules:
    • cryptography
    • httpx[http2]
    • Optional: streamlink
  • External Dependencies:
    • ffmpeg and ffprobe

Installation Steps:

  1. Download the source code.
    2. git clone https://github.com/cabernetwork/cabernet
  2. Navigate to the cabernet directory.
    3. cd cabernet
  3. Install Python dependencies.
    4. pip3 install -r requirements.txt
  4. Launch the app by running: 
    python3 tvh_main.py
    This will create a data folder and a config.ini file inside it.
  5. Open your browser and navigate to: http://your-ip-address:6077/
  6. Install plugins from the Plugins tab
  7. Restart Cabernet via: 
    Scheduled Tasks > Applications > Restart
  8. Customize your settings:
    • Logging: Adjust log levels (e.g., from "warning" to "info") if needed.
  9. Test the functionality using the XML/JSON links available in the app.
  • NOTE: Manual installation may work on Windows or macOS, but is untested. We reccomend to use the installer or Docker.

    • Docker

    Available Architectures:

    Architecture Supported
    x86-64 ✅
    arm64 ✅
    armhf ❌

    Note: Ensure the volume for /app/.cabernet is set before enabling encryption. ARMHF is not supported as Python's cryptography module only works on 64-bit systems.

    Installation Options:

    a. Using docker-compose:

    1. Download the Cabernet source and unpack it into a folder.
    2. Edit the docker-compose.yml file to set volume folder locations.
    3. Pull the Cabernet image: 
      docker-compose pull cabernet
    4. Start the container: 
      docker-compose up -d cabernet

    b. Using docker CLI:

    docker run -d \
--name=cabernet \
-e PUID=1000 \ # optional
-e PGID=1000 \ # optional
-e TZ=Etc/UTC \ # optional
-p 6077:6077 \
-p 5004:5004 \
-v /path/to/cabernet/data:/app/data \ # optional
-v /path/to/plugins_ext:/app/plugins_ext \ # optional
-v /path/to/cabernet/secrets:/app/.cabernet \ # optional
--restart unless-stopped \
ghcr.io/cabernetwork/cabernet:latest

    Parameters:

    Parameter Function
    -p 6077 Cabernet WebUI
    -p 5004 Stream port
    -e PUID=1000 User ID
    -e PGID=1000 Group ID
    -e TZ=Etc/UTC Timezone (use this list)
    -v /app/data Stores database and configuration files
    -v /app/plugins_ext External plugins
    -v /app/.cabernet Encryption key storage

    Additional Notes:

    Set Clients > Web Sites > plex_docker_ip in the Cabernet configuration to your computer's IP address (not the internal Docker container IP). This ensures the channels.m3u file points to the correct IP for streaming.

    Updating container:

    Using docker-compose:

    1. Stop and remove the current container: 
      docker-compose rm --stop -f cabernet
    2. Pull the latest image: 
      docker-compose pull cabernet
    3. Restart the container: 
      docker-compose up -d cabernet

    Using docker CLI:

    1. Pull the latest image: 
      docker pull ghcr.io/cabernetwork/cabernet:latest
    2. Stop the running container: 
      docker stop cabernet
    3. Remove the container: 
      docker rm cabernet
    4. Clean up old images (optional): 
      docker image prune

    Using Watchtower Auto-Updater:

    docker run --rm \
-v /var/run/docker.sock:/var/run/docker.sock \
containrrr/watchtower \
--run-once cabernet

    For continuous updates, follow Watchtower instructions.
